Proyek ini berisi hasil praktikum konfigurasi VLAN dan Trunking pada perangkat Switch Cisco menggunakan Cisco Packet Tracer. Praktikum ini bertujuan untuk memahami cara melakukan segmentasi jaringan dengan VLAN, melakukan konfigurasi port, mengelola database VLAN, serta membangun konektivitas antar-switch melalui trunking. Pada kegiatan ini dilakukan beberapa tahapan, dimulai dari membangun topologi jaringan dasar, melakukan konfigurasi dasar pada switch seperti pemberian hostname, password, banner, hingga pengaturan IP management pada VLAN. Selanjutnya dibuat beberapa VLAN seperti VLAN 10, 20, 99, dan 1000 sebagai jaringan tersegmentasi sesuai fungsi tertentu. Port-port pada switch kemudian dipindahkan dari VLAN default (VLAN 1) ke VLAN baru yang telah dibuat, serta dilakukan pengelolaan VLAN dengan memindahkan port antar VLAN, menghapus VLAN tertentu, hingga membuat VLAN secara otomatis melalui konfigurasi port.

Setelah itu, koneksi antar-switch dikonfigurasi melalui trunk dengan menggunakan mode Dynamic Trunking Protocol (DTP) maupun trunk manual menggunakan 802.1Q. Pengujian koneksi dilakukan menggunakan perintah ping untuk memastikan apakah komunikasi antar VLAN berjalan sesuai konfigurasi. Hasil pengujian menunjukkan bahwa PC dalam VLAN yang sama dapat saling berkomunikasi setelah trunk aktif, sementara komunikasi antar VLAN tidak dapat dilakukan tanpa perangkat Layer 3 karena switch Layer 2 tidak mendukung routing antar VLAN. Melalui konfigurasi ini juga dipahami bahwa penggunaan VLAN 1 sebagai native VLAN tidak disarankan karena kurang aman, sehingga native VLAN diubah menjadi VLAN 1000 untuk meningkatkan keamanan jaringan.

Secara keseluruhan, praktikum ini memberikan pemahaman mengenai bagaimana VLAN bekerja dalam memisahkan jaringan, bagaimana port pada switch dipetakan ke tiap VLAN, dan bagaimana trunking memungkinkan VLAN yang sama tetap terhubung meskipun berada pada switch yang berbeda. Selain itu, penggunaan perintah seperti show vlan brief dan show interfaces trunk menjadi langkah penting dalam memverifikasi hasil konfigurasi yang telah dilakukan.
